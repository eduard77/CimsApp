## 1. WBS PRINCIPLES

- Decomposed to the **work package level** вЂ” each can be estimated, scheduled, and assigned to a single owner
- **100% rule** вЂ” the WBS represents 100% of project scope; no more, no less
- Each element has a **unique code** following the hierarchy
- **Deliverable-oriented** вЂ” decomposed around outputs, not activities

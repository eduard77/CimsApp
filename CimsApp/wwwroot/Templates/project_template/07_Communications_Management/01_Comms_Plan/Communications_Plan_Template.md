## 3. REPORTING STANDARDS

### 3.1 Progress Report — Monthly (structure)
1. Executive Summary
2. Safety & Environment
3. Progress against programme (milestones + SPI)
4. Financial status (cost + CPI + cashflow)
5. Quality (NCR status + audit findings)
6. Risk & issues (top 10)
7. Key decisions required
8. Look-ahead (next month)
9. Photos & evidence

### 3.2 Document Naming Convention (ISO 19650-2)
```
[Project]-[Originator]-[Zone]-[Level]-[Type]-[Role]-[Number]
e.g.  PRJ001-ABC-Z1-L02-DR-A-0001
```

### 3.3 CDE Workflow
All project information flows through the four ISO 19650 states:
`WIP → SHARED → PUBLISHED → ARCHIVED`

## 4. ESCALATION PROCEDURE

| Level | Response Time | Escalate To |
|---|---|---|
| Routine issue | 1 working day | Line manager |
| Significant issue | Same day | Project Manager |
| Major issue | 2 hours | Sponsor |
| Crisis (HSE/legal/reputational) | Immediate | Steering Group + Executive |

---

## 5. INFORMATION SECURITY (ISO 19650-5)

| Classification | Who can access | How distributed |
|---|---|---|
| OFFICIAL | All project team | CDE — standard |
| SENSITIVE | Named project members | CDE — access-controlled folder |
| RESTRICTED | Named individuals | CDE — encrypted + audit log |
| CONFIDENTIAL | Sponsor + approved | Out-of-band; no CDE storage |

## Decisions required at kickoff (lower-stakes than S10/S11)

**Decision 1 — PDCA state-machine shape.**

Plan-Do-Check-Act is a continuous improvement cycle. Two v1.0
shapes:

- **A (recommended):** 5-state state machine in
  `Core/PdcaWorkflow.cs` — Plan → Do → Check → Act → Plan
  (cycle back for next iteration) | Closed. Each transition
  records the actor + timestamp + per-stage notes. Honest
  PDCA workflow.
- **B:** 2-state Open/Closed with PDCA stage as a tag rather
  than enforced workflow. Simpler to implement; weaker as a
  "PDCA register" claim.

**Default at kickoff: Option A.** The state machine is now
near-zero estimate (S6 retro action, 5-for-5 across S5/S6/S10/
S11). Pattern reuse from DpiaWorkflow.

**Decision 2 — Cross-project lessons library scoping.**

The F.12 second bullet says "tenant-scoped". Two
interpretations:

- **A (recommended):** Org-scoped — any authenticated user in
  the tenant can read the lessons library. The `cross-project`
  modifier means "lessons cross project boundaries within the
  tenant", not "per-project visibility".
- **B:** Per-project — only project members can see lessons
  recorded against that project.

**Default at kickoff: Option A.** Cross-project is the whole
point; restricting to project membership defeats the bullet.

**Decision 3 — OpportunityToImprove source linkage.**

The F.12 third bullet says "linked from any module". Two
v1.0 shapes:

- **A (recommended):** JSON metadata pointer
  (`SourceEntityType` string + `SourceEntityId` Guid) on
  OpportunityToImprove. Free-text discoverable; v1.1 / B-083
  upgrades to a structured polymorphic FK (per-domain join
  tables).
- **B:** Polymorphic FK now — per-domain join tables for
  every module. Massively more work; touches every entity.

**Default at kickoff: Option A.**

## Top 3 risks (PAFM-SD Appendix E)

1. **OpportunityToImprove source-linkage type drift.** v1.0
   uses a free-text SourceEntityType string; nothing
   prevents typos ("Risk" vs "risk" vs "RISK"). **Mitigation:**
   service-layer normalisation to PascalCase + a per-tenant
   distinct-values endpoint to surface what's already in use
   (caller can pick from the live set). Free-text invariants
   are the v1.0 cost.
2. **PDCA cycle-back semantics.** Act → Plan (next iteration)
   vs Act → Closed are two distinct decisions. **Mitigation:**
   the state machine offers Act → Plan AND Act → Closed as
   separate transitions; UI / caller picks. Cycle counter +
   per-cycle history is B-081.
3. **Lesson library size growth.** Org-scoped + cross-project
   means lessons can accumulate fast. **Mitigation:** v1.0
   simple list endpoint with optional `?since=` and `?tag=`
   filters; full-text search → B-082.

can manually create / track / complete inspection activities

## Decision required at kickoff: PAFM Ch 47 unblock

PAFM Ch 47 (Genera Systems QA, HS&E integration specification)
is paste-on-request from `Notes/PAFM_v2.0_complete.pdf` per
the `reference_pafm_spec.md` discipline. Without it, three
plausible v1.0 paths:

- **Option A (recommended, decided at kickoff): defer the
  integration bullets.** v1.0 ships only the
  `InspectionActivity` entity + service + manual-entry
  endpoints. The four F.13 integration bullets become v1.1
  / B-086..089 explicitly (Ch 47 paste + Genera-side API
  reciprocity is the unblock condition).
- Option B: ship a stub integration layer with placeholder
  schemas. Adds complexity for zero v1.0 value because the
  placeholder schema is almost certainly wrong vs Ch 47;
  customers can't use a stub.
- Option C: pause S13 until Ch 47 is pasted. Time is
  unspent; sprint cadence breaks.

**Default at kickoff: Option A.** Ship the data shape
honestly; defer the integration explicitly with B-NNN
unblock conditions. Pilot users still benefit (manual
inspection tracking in CIMS); v1.1 wires up Genera when
Ch 47 lands.

## Top 3 risks

1. **Ch 47 may arrive mid-pilot, not mid-S13.** If Ch 47
   pastes during pilot rather than during a future v1.1
   sprint, B-086..089 may need to land as a fast-follower
   sprint rather than the planned v1.1 sequence.
   **Mitigation:** v1.0 InspectionActivity entity is shaped
   as the canonical CIMS-side data; whatever Ch 47 says
   about Genera-side wire format, the CIMS data model is
   stable. Wire-format reconciliation is a v1.1 mapper
   layer, not a re-modelling.
2. **Manual-entry workflow may surprise pilot users who
   expect Genera integration.** **Mitigation:** name the
   v1.0 endpoint set clearly (e.g.
   `/api/v1/projects/{projectId}/inspection-activities`);
   document in the role-matrix that "Genera sync — v1.1".
3. **InspectionActivity state machine may need re-shape
   when Ch 47 arrives.** **Mitigation:** v1.0 ships the
   simplest reasonable shape (Scheduled → InProgress →
   Completed | Cancelled). If Ch 47 needs more states
   (e.g. AwaitingApproval, ReportedNonCompliant), v1.1 /
   B-086 extends the enum; existing rows migrate by being
   in their current state.

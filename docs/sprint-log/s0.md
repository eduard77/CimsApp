data layer, audit captures actor / action / entity / before / after, and

## Top 3 risks (PAFM-SD Appendix E)

1. **Applying the query filter may break existing queries.** Seed data,
   background migrations, and any cross-tenant admin code will fail
   once the filter lands. Mitigation: add filter to entities one at a
   time; full build + test after each.
2. **`ITenantContext` DI lifetime interacts awkwardly with `DbContext`.**
   Both are scoped; if the context is read during `OnModelCreating`
   (model build, singleton cache) it'll break. Mitigation: read
   `ITenantContext` at runtime in the filter expression via a captured
   `DbContext` field or a `EF.Property`-safe pattern.
3. **Existing controllers may be unit-tested against data that the new
   filter hides.** Tests that seeded rows without an OrganisationId or
   with a fake one will fail. Mitigation: seed test data with a
   consistent `OrganisationId` and wire the test `ITenantContext`
   accordingly.

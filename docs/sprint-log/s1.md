**Planned:** 2026-04-26 → 2026-05-17 (3 weeks)
**Roadmap reference:** PAFM-SD Ch 6.3, Phase 2
**Module DoD:** PAFM Appendix F.2

## Sprint goal

Deliver the Cost & Commercial module (PMBOK 8 Finance domain) on
the foundation laid by Sprint 0. Covers cost breakdown structure
(CBS), earned value management (EVM), variations / change orders,
payment certificates, and cashflow forecasting in the UK
construction context (NEC4 / JCT contractual conventions).

Inherits from S0:
- `ITenantContext` and the full query-filter scoping (every Cost
  entity is tenant-scoped via its `Project.AppointingPartyId`).
- Two-tier role model (ADR-0010): financial endpoints are
  candidates for `[Authorize(Roles=...)]` global gates plus
  per-project `HasMinimumRole` checks.
- Audit interceptor — every cost / variation / certificate write
  is automatically audited.
- Invitation-token registration so only authorised users land in
  a tenant.

## Top 3 risks (PAFM-SD Appendix E)

1. **NEC4 / JCT semantics are easy to get subtly wrong.** Payment
   certificate calculations differ between the two conventions
   (e.g. how retention is handled, when the activity-schedule
   resets). Mitigation: pick one convention as the v1.0 default
   (likely NEC4 given the construction sector), document the choice
   in an ADR, defer the alternative to v1.1.
2. **EVM rollup performance on large CBS trees.** A real construction
   project has 200-500 CBS items. Naïve N+1 queries on rollup will
   crawl. Mitigation: use EF Core's `Include` / projection from
   day 1, behavioural-test the rollup with a 500-item fixture
   before any UI work, profile.
3. **Variations workflow scope creep.** Real-world change-order
   processes have 5-10 stages including legal review, programme
   impact assessment, etc. Mitigation: deliver a 3-state machine
   (Raised → Approved/Rejected) for v1.0 and explicitly defer
   intermediate stages to v1.1 backlog with a change record if
   needed.

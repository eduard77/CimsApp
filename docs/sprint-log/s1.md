## Top 3 risks (PAFM-SD Appendix E)

1. **NEC4 / JCT semantics are easy to get subtly wrong.** Payment
   certificate calculations differ between the two conventions
   (e.g. how retention is handled, when the activity-schedule
   resets). Mitigation: pick one convention as the v1.0 default
   (likely NEC4 given the construction sector), document the choice
   in an ADR, defer the alternative to v1.1.
2. **EVM rollup performance on large CBS trees.** A real construction
   project has 200-500 CBS items. NaĂŻve N+1 queries on rollup will
   crawl. Mitigation: use EF Core's `Include` / projection from
   day 1, behavioural-test the rollup with a 500-item fixture
   before any UI work, profile.
3. **Variations workflow scope creep.** Real-world change-order
   processes have 5-10 stages including legal review, programme
   impact assessment, etc. Mitigation: deliver a 3-state machine
   (Raised â†’ Approved/Rejected) for v1.0 and explicitly defer
   intermediate stages to v1.1 backlog with a change record if
   needed.

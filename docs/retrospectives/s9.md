## Retrospective

### What went well

- **The kickoff audit "what's already shipped" framing was
  the right call.** F.9's five DoD bullets read as "build
  this whole module" but three of the five were already
  done. By calling that out at kickoff, the sprint focused
  estimate + risk on the genuinely-new work (MIDP / TIDP)
  and kept the audit-ticking work (T-S9-04) as a small
  closing task rather than a from-scratch re-build. **Sprint
  closed in one session at 30h estimate (vs 72h capacity)
  because the audit-vs-build distinction was honest.**
- **The S8 CI hardening jobs paid off on their first sprint
  exercise.** Before pushing the sprint branch I expected
  CI to run; what actually happened is the SQL Server smoke
  job validated my new AddMidpTidp migration applies cleanly
  on real SQL Server, and the migration round-trip job
  validated it reverts cleanly. Both passed first push.
  Without S8 the sprint would have closed with "tests pass
  locally, CI is just unit tests". With S8 the sprint closes
  with "tests pass locally + CI exercised the migration on
  real SQL + migration round-trip proved Down() works on the
  new entities." The infrastructure investment from S8 is
  paying interest.
- **B-NNN inline discipline now feels routine — fourth
  consecutive sprint.** B-065..B-069 landed in T-S9-05/06's
  same commit. Closing audit: 0 entries. The S5-retro rule
  has fully embedded.
- **The reconciliation surface was real but bounded.** The
  parked Iso19650ReferenceData.cs vs Core/Iso19650Codes.cs
  divergence had two genuine differences: A-codes (Core
  has them, parked didn't) and Volume/Level coding (Core
  uses Annex A whitelists, parked used 2-digit regex). Both
  surfaced as test failures or test rewrites; neither
  required a code rewrite. ~3 lines of validator change to
  fix the Volume/Level regex; 1 test rewrite for A-codes.
  The "reconciliation might be a rabbit hole" risk
  evaluated honestly at kickoff was avoided.
- **The B-027 broken-Down() insurance from S8 paid off
  preemptively here.** When AddMidpTidp generated, I trusted
  it would round-trip without verifying because S8's
  migration-round-trip CI job runs on every push. Pre-S8
  I might have either (a) tested it manually with a separate
  dotnet ef step or (b) deferred verification to "we'll
  notice if it breaks". S8's CI made (a) automatic. That's
  exactly the value the kickoff Top-3 risk #3 (broken Downs)
  was designed against.

### What did not go well

- **The `MD` → `M3`/`M2` test fixture rewrite I'd predicted
  at kickoff turned out to be unnecessary.** Tests didn't
  reference `MD` directly. The kickoff Top-3 risk #1
  ("reference-data merge regression") spent budget on a
  predicted rewrite that didn't materialise. A related but
  different rewrite WAS needed (Suitability_check_fails_on
  _A_code_in_v1). Honest finding: the kickoff risk was
  pointed at the wrong specific test. The category of risk
  ("reference-data merge surfaces test rewrites") was
  correct, just not the specific test. Worth recording — at
  S9-style reconciliation sprint kickoffs, predict the
  CATEGORY of risk, not the specific lines.
- **Volume / Level regex bug was honest dev oversight, not
  a S0 leftover.** The pre-S9 validator's `^\d{2}$` regex
  was added when the parked validator was first written
  (sessions 1-2), pre-Core/Iso19650Codes — i.e. before
  there was a canonical broader code set to validate
  against. Switching to Hashed whitelists is correct, but
  this is the first time anyone (validator or app) actually
  exercised that path against the canonical Annex A set.
  In a fully-test-driven flow this would have been caught
  by writing the validator-against-Core test first. Not a
  fault, but a lesson: when reconciling two reference-data
  sources, always run the broader source through the
  consumer's tests as the FIRST reconciliation step.

### What surprised me

- **MIDP + TIDP collapsed to 2 entities, not 4.** I'd half-
  expected a header-plus-entries shape (MIDP header +
  MidpEntry collection + TIDP header + TidpEntry collection).
  Once I started writing the entities, the header rows had
  no fields — they'd just be Project FK + Id. Collapsed to
  MidpEntry + TidpEntry directly. ~50% less entity surface
  for the same domain coverage. Worth recording as a
  micro-pattern: when a domain object's "header" entity has
  only FK fields, collapse to the collection.
- **All 11 MIDP / TIDP endpoints worked first try (after
  the migration).** No bugs found by tests during writing;
  17 tests passed first try. Honest finding: the patterns
  from S5/S6/S7/S8 (CRUD + transitions + soft-delete + audit-
  twin + cross-tenant 404) are now so internalised that
  re-applying them to a new domain is mostly typing.
- **Text count delta of +30 was honest for the sprint
  shape.** Reconciliation surface added very few tests (1
  rewrite, 1 new); MIDP / TIDP added 17; coverage tests
  added 4; tenant sweep added 1; integration tests added 7.
  Lower than S6's +149 (full workflow domain) or S7's +37
  (read shape) — about right for a "two new entities + bunch
  of audit-by-reference work" shape.

### What I will try differently next sprint

- **Predict risk categories at kickoff, not specific test
  rewrites.** S9's kickoff Top-3 risk #1 was pointed at the
  `MD` → `M3`/`M2` rewrite, which didn't fire. The actual
  rewrite that fired (A-codes test) was in the same risk
  category. Future kickoffs: state the category and a
  one-line example, don't lock in a specific test name.
- **For reconciliation-shaped sprints, run the broader
  source through the consumer's tests FIRST.** Pre-S9 the
  validator was using its own narrower reference data; the
  Volume/Level regex bug only surfaced when the broader
  Core source was wired in. The micro-discipline: when
  swapping reference data, run the consumer's full test
  suite immediately after the swap, not after the wiring.
- **Re-examine S0 entities for "header-with-only-FKs"
  collapse opportunities.** MIDP/TIDP collapsed cleanly;
  it's worth a one-pass audit of the existing entity model
  to see if anywhere else has a vestigial header. Probably
  none, but worth checking. Audit task — not a v1.0 blocker.

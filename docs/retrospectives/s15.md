## Retrospective

### What went well

- **Triple Option A held cleanly through to close.** Backend
  (LIKE), endpoint shape (cross-entity), saved-search
  (deferred) — none re-litigated. Triple-Option-A pattern
  now empirically 5-for-5 (S10 / S11 / S12 / S14 / S15).
- **Scope-spec caveat surfaced, accepted, held.** The .docx
  bullets weren't loaded; instead of guessing silently or
  blocking the sprint, the kickoff doc opened with a
  caveat and the user picked Option (a) — proceed with the
  predicted shape. Pattern to carry: when scope is
  uncertain and the user is available, **ask once with a
  concrete proposal** and proceed under their answer rather
  than waiting for a perfect spec read.
- **EF DbContext thread-safety hazard caught at design time.**
  Initial draft of the aggregator used `Task.WhenAll` over
  per-provider tasks — would have thrown "second operation
  started before previous completed" the first time two
  providers tried to read concurrently. Caught while
  reviewing the draft and switched to sequential await
  before any test ran. The compile-then-run loop would have
  caught it too, but design-time recognition is cheaper.
  Generalisable: **whenever a service fans out via
  IServiceProvider over a scoped EF context, the fan-out is
  sequential, not parallel.**
- **LIKE wildcard escape was a design decision, not an
  oversight.** Top-3 risk #1 named the hazard at kickoff;
  the helper was written before any provider; one test
  exercises every wildcard character explicitly. Previous
  sprints' Top-3 predictions paid off similarly; this is a
  reusable pattern.

### What did not go well

- **Same `Risk` namespace conflict from S14.** First test
  build failed with `'Risk' is a namespace but is used like
  a type` — same root cause as the S14 T-S14-04 build break
  (the `CimsApp.Tests.Services.Risk` namespace from the
  `RisksServiceTests` folder shadows `CimsApp.Models.Risk`).
  S14 retro said "read entity definitions before writing
  test fixtures" — the right lesson but not the one that
  would have caught this. The actual lesson: **when the
  test project's folder structure creates a namespace that
  collides with a model type, fully-qualify the type at the
  call site or restructure the namespace.** ~2 minutes lost.
  Worth carrying forward as a permanent code-shape note.
- **+7 tests is the lowest of any feature sprint since S0.**
  By design — search is read-only, no entity, no migration,
  no state machine. The seven tests cover the surface
  comprehensively (escape helper, aggregation, isolation,
  types-filter, min query length, per-provider sanity).
  Pre-pilot QA may want more wildcard / Unicode / edge-case
  tests; flag as a v1.1 fast-follower rather than a
  separate B-NNN.

### What surprised me

- **Sequential fan-out is fast enough.** Seven LIKE queries
  through indexed columns with project-id pre-filter run
  in single-digit milliseconds each on the in-memory test
  provider. Real SQL Server with a populated project will
  be slower but the order-of-magnitude is the same. The
  performance story stays "fine for v1.0 pilot, FTS for
  scale" — exactly what kickoff predicted.
- **`Task.WhenAll` over scoped `DbContext` is a recurring
  trap.** S14's hosted services dodged it (per-iteration
  scope); S15 hit it head-on (single scoped context fanned
  out across providers). Worth promoting to the discipline
  manual: "Scoped `DbContext` + `Task.WhenAll` = exception".

### What I will try differently next sprint

- **Carry "namespace-collision in tests" as a permanent
  pattern memo.** Two consecutive sprints have been bitten
  by the same shape; deserves an explicit note in the test
  conventions rather than relying on retro-discovery each
  time.
- **S16 scope decision.** Per the post-S8 renumbered roadmap
  the next default lean is **F.16** — exact bullet unknown
  pending the same .docx-paste reconciliation that S15
  ducked. Likely candidates: F.16 Mobile / Responsive Views
  (per pre-renumber S0..S7 retros) OR F.16 Admin Console
  (per S3 enum-comment hints). Surface the same caveat
  pattern in S16 kickoff: predicted bullets, ask once, hold
  to the answer.
